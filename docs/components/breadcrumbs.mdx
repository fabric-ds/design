# Breadcrumbs

Breadcrumbs show hierarchy and navigational context for a user’s location within
an app.

## Options

<Guideline title="Default" image="breadcrumb-options-default.svg">
  By default, breadcrumbs are displayed inline with the hierarchy shown in
  reading order.
</Guideline>

## Behaviors

<Guideline title="Overflow" image="breadcrumb-behaviors-overflow.svg">
  Breadcrumbs truncate when there is not enough room to display all levels of
  the breadcrumb list, or as a way of managing relevance of the visible
  breadcrumb items in a deeply nested hierarchy. Always truncate at the top level, on the left, so that the current level and as many possible parent levels are visible.
</Guideline>

## Usage guidelines

<Guideline title="Use breadcrumbs to show hierarchy" image="breadcrumb-usage-guidelines-1-do.svg" good>
  Breadcrumbs need a consistent hierarchical structure because they create a
  path for discovery and context for a user’s current location.
</Guideline>

<Guideline title="Icons in breadcrumbs" image="breadcrumb-usage-guidelines-2-dont.svg" bad>
  Don't use icons within breadcrumbs. Since breadcrumbs are horizontally
  distributed, icons disrupt the rhythm and readability of the list.
</Guideline>
