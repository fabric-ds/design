# Alert

Alerts display a non-modal message associated with objects in a view. These are used to inform the user of information relevant to the task they're trying to complete.

## Options

<Guideline title="Informative variant" image="alert-options-informative.svg">
  The informative variant uses the informative semantic color (blue) and has an "information" icon to help those with color vision deficiency discern the message tone.
</Guideline>

<Guideline title="Positive variant" image="alert-options-postiive.svg">
The positive variant uses the positive semantic color (green) and has a "checkmark" icon to help those with color vision deficiency discern the message tone. This should be used to inform someone of a successful function or result of an action they took.
</Guideline>

<Guideline title="Warning variant" image="alert-options-warning.svg">
The warning variant uses the notice semantic color (orange) and has an "alert" icon to help those with color vision deficiency to discern the message tone. This can be used to warn about a situation that may need to be addressed soon.
</Guideline>

<Guideline title="Negative variant" image="alert-options-negative.svg">
The negative variant uses the negative semantic color (red) and has an "alert" icon to help those with color vision deficiency to discern the message tone. This can be used to show an error or a failure, or to convey something that needs to be immediately acknowledged or addressed.
</Guideline>

## Behaviors

<Guideline title="
Text overflow" image="alert-behaviors-text-overflow.svg">
When the title of an alert is too long for the available horizontal space, it wraps to form another line. The icon stays aligned to the top.
</Guideline>
